## Context

The site needed a notes section. The obvious option was another Astro content
collection reusing `Layout.astro` and the blog's table-of-contents sidebar —
about four files, no new dependencies, visually identical to the rest of the
site.

## Decision

Use Starlight, and upgrade Astro 6 → 7 to get it.

Accepted that `/docs` looks different from the rest of the site. Starlight owns
the header, sidebar and theme there; no custom styling was attempted.

## Consequences

**Cost paid up front:**

- Astro 6.4.8 → 7.1.3, dragging Vite 6 → 8 and the new Sätteri markdown pipeline
  (see [0002](/docs/decisions/0002-astro-7/)).
- Expressive Code can no longer be its own integration — Starlight bundles it and
  the two conflict. Its config moved inside `starlight({ expressiveCode: … })`.
  Starlight registers it site-wide, so blog code blocks kept their theme.
- Starlight's 404 route silently replaced `src/pages/404.astro`. Needed
  `disable404Route: true`.
- Content sits at `src/content/docs/docs/` — one extra level — because Starlight
  serves `src/content/docs` at the site root and has
  [no subpath support](https://github.com/withastro/starlight/issues/347).

**What was bought:** a real docs sidebar, autogenerated from folders, with zero
layout code to maintain. Adding a section is creating a directory.

**Revisit if** the visual split starts to grate, or Starlight's upgrade cadence
becomes a burden on the whole site rather than just `/docs`.
